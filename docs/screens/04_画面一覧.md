## 3. 画面遷移図

### 3.1 医師の典型的な操作フロー

```
[C-001 ログイン]
  ↓
[C-003 ダッシュボード] ← ホーム
  ↓
[M-002 待合室一覧] ← 診察待ち患者確認
  ↓
[P-001 患者検索] ← 患者選択
  ↓
[P-002 患者基本情報] ← 患者情報確認
  ├→ [P-005 アレルギー情報] ← アレルギー確認
  ├→ [P-006 既往歴] ← 既往歴確認
  └→ [M-011 バイタルグラフ] ← バイタル確認
  ↓
[M-004 カルテ一覧] ← 過去カルテ確認
  ↓
[M-005 カルテ記録（SOAP）] ← カルテ記載
  ├→ [M-008 診断名登録] ← 病名登録
  ├→ [O-001 処方オーダー] ← 処方
  │   └→ [O-002 処方履歴] ← 過去処方参照
  ├→ [O-005 検査オーダー] ← 検査依頼
  │   └→ [O-007 検査結果一覧] ← 過去検査参照
  └→ [R-002 予約受付] ← 次回予約
  ↓
[M-012 電子署名] ← カルテ署名
  ↓
[C-003 ダッシュボード] ← 次の患者へ
```

### 4.2 画面要素の配置ルール

1. **ヘッダー部**:
   - システム名、ログインユーザー名、現在日時
   - ログアウトボタン
   - アラート・通知アイコン

2. **ナビゲーション部**:
   - 主要機能へのグローバルメニュー
   - パンくずリスト

3. **サイドメニュー部**:
   - サブメニュー
   - クイックアクセス

4. **メインコンテンツ部**:
   - 画面タイトル
   - 検索条件エリア（検索画面の場合）
   - データ表示エリア
   - アクションボタンエリア

5. **フッター部**:
   - コピーライト
   - ヘルプへのリンク

## 7. 画面別必須項目チェック

各画面で必須入力項目を明確化し、未入力時はエラーメッセージを表示。

**例: P-003 患者新規登録画面**
- 必須項目: 姓、名、姓カナ、名カナ、生年月日、性別、患者番号（自動採番可）

**例: M-005 カルテ記録画面**
- 必須項目: 患者ID、来院ID、記録日時、記録者、SOAPまたはフリーテキストのいずれか

**例: O-001 処方オーダー画面**
- 必須項目: 患者ID、薬剤名、用量、用法、日数

## 8. エラーメッセージ表示方針

1. **入力エラー**: フォーム上部に赤枠でエラーメッセージ表示
2. **システムエラー**: モーダルダイアログで表示
3. **警告**: 黄色背景で注意喚起
4. **アレルギーアラート**: 赤色で強調表示、音声アラート
